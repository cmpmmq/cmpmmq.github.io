在经过一通配置后，可以用 md 直接写文章，然后 post 到网站上，可以说是很有成就感了。网站搭建的过程和心得就如下文了，第二篇博客还写了挺多字的。

为什么写个人博客 | Why blog

  1. 个人网站的优越性：社交媒体是一种主动展示吧，个人网站是一种被动展示，属于如果你想看，你可以点进我的网页，但不是我突然 post 在 pyq，然后让你点进来的，有点“愿者上钩”的感觉。
  2. 我以前初中和高中写过两次微信公众号，现在账号密码都忘了，但被好多认识的人关注实在很社死。但谁让我又喜欢写呢哈哈。
  3. 高考心愿 to-dos ：在不以分数为目的指导下，学自己喜欢的东西，做自己喜欢的事，搭建一个网站，在这个过程中学习编程。
  4. 在自己的网站上放项目总结，编程、跑步、旅游、读书等文章，如果录取了新闻系还可以repo采访；甚至可以在自己网站上连载小说，typora 可以用 gitbook 生成书。

如何搭建 | How to set up

稍微回忆一下整个过程

只是复盘自己摸索的过程，并不是标准写的一篇好用的教程。

  1. 注册 github 账号，并将 repository 仓库命名为 “昵称.github.io”，要设置为公开 public，然后该网站已经可以运行。
  2. 在本地使用 visual studio code 作为编译环境。
  3. 在 mac 环境下，用镜像先安装 homebrew，然后用 Terminal。
  4. 执行命令安装 git，配置用户名、密码、SSH keys。测试 git comment, clone, push是否能正常使用。
  5. 这时候甚至可以从 html 开始完全写一个网页，然后以后自己用 css 渲染，加上 js 可以在网页上跑程序。可以实时编辑，也挺有成就感的，但我弃坑了，这是以前我的网站。很简陋，所以还是 hexo 香。样式、功能都有，各个版本都有适配。
  6. 安装 hexo。我是后来才看见的这个文档，非常好用，可以照做，也有视频 easyhexo。
  7. 安装-配置-部署，都在上面的网站有。配置在本地的 _config.yml 文档，然后部署到自己的 GitHub Pages。
  8. 安装主题，选一个官方文档写得好的很重要！那样的话直接照做就行了。直接 git clone 然后在配置文件中把 themes 改掉就行。剩下的稍微个性化一些的配置要看官方文档。看上面 easyhexo 推荐的很多个主题，或者使用我现在用的 Keep Theme。
  9. 在本地装个好点的 markdown 编辑器吧，Typora 就很好，开源还免费。

Start Blogging!!

我在这个过程中学会了什么

  1. 如何使用mac。买新电脑后激励自己学点新东西，结果一开始连开关机都不会，键盘、鼠标手势、快捷键也要全部从头开始学。
  2. git 其实还是不大会，就是看着别人怎么做，然后自己贴一下代码，然后就装好了。不知道为什么跑起来了，也不知道为什么跑不起来。
  3. 装了 vs code，大学里总归用得上的，大学编程不在话下（bushi）。
  4. 锻炼百折不挠、越挫越勇的精神，通过 b站和百度搜索照着做，硬着头皮看报错，有时候半懂不懂还接着做吧。印象里有一次本地里网站跑起来了， deploy 上去出错，自闭了，过了一会又好了，大概因为 github 太慢了。所以要对自己有信心吧哈哈，实在不行就过几天再接着 debug。
  5. 其实有些人的教程做得有点坑，我 b站上录的第一个教程也不好。可能会重新做一次吧，录教程和写博客是当作总结和反思的步骤，在教别人的过程中学习。
  6. 熟悉 markdown 的语法，markdown 我一直用的，其实也不复杂，就是现在要用一下代码块。比如说安装 hexo-browsersync  插件，就不用手动刷新，只要保存就会自动刷新页面看效果，另外这里演示用到了链接和代码块的 md 语法。其他的在写作中用加粗，标题（自动生成目录 toc），斜体，引用，完全够用了。我要演示 md 语法的话要用转义符啊，那下次填坑吧。
  1. 跟着 Keep Theme 做的，然后有讲到网页 load 的加速， 并且使用了 PicX 图床。
  2. 写代码一时爽，一直写代码一直爽。

新坑 | New Ideas

  1. 实现多平台同步，iOS全家桶下使用 icloud 的可行性。win+andriod 用坚果云？
  2. 加上 Gitalk 的插件，虽然只有用 Github 的人能来评论～
  3. 用一下 gitbook，导入一些以前写的文章，总结一个项目。
  4. 写个镜像的 English 版本的，每写一篇文章还自己翻译一下，练习英语塞。
  5. 加上更多功能，比如统计文章篇目、总字数，文章内加音乐、b站视频，hexo其他插件的尝试等等。
  6. 在 posts 里面可以新建 .html ，应该可以设置不同的css，再运行 js 程序；我记得是可以新建一个文件夹的。
  7. 在上述实践基础上，学习前端的更多知识，尝试写个自己的主题。
  8. 既然写了hexo的主题，那也可以写个 typora 的皮肤。
  9. 坚持写 blog，网页上作者边上的 lv1,2,3 … 是以文章数量为评判的，所以先多写点，然后升级。如果自己坚持下来了，在这个过程中锻炼了写作和编程维护博客的能力，就买个腾讯云域名之类的。
  10. 好像这样之后可以集成，说是发布更加简单一些。
  11. tag 功能的升级，多个 tag 的筛选，取交集、并集、补集。要找到主题的 js 代码，然后这个功能在我写了足够多的 blog 下才有意义。
  12. 想加个彩蛋和隐藏入口，是 tag 的功能。
  13. 甚至还有个 RSS 订阅功能可以开启。
  14. 提速，现在的局限是服务器，以后文章多了也许会有新的问题，不过现在已经用图床 PicX 加速了。02
